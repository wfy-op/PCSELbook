# Review 意见（修订版，按“教材终审”标准）

## 总评

这版**明显比上一版更强**。最关键的进步有三点：

1. **主线更稳了**：第二章把层级、术语、红线、依赖图都立住了，读者不容易再一开始就混淆 `Q / g_th / I_th`。
2. **workflow 更像教材了**：第二十五章和第二十六章已经不是“研究者备忘录”，而是在尝试把建模流程变成可教的程序。
3. **器件侧接口更明确了**：第十七至二十章和第二十九、三十章已经把“光学结果不能直接冒充器件结论”这件事说得足够清楚。

但如果标准是：**一个只有本科物理基础的读者，只靠这一本书，就能从 0 建立 PCSEL 的理论与仿真工作流，并且不至于在第一次独立建模时系统性犯错**，那么这版**仍然没有完全过线**。

当前最大的问题已经不是“缺知识点”，而是：

- **有些地方仍然是正确地提醒，而不是完整地教会**；
- **有些地方仍然是工作流模板，而不是 fully worked example**；
- **有些地方宣称 rigor，但 rigor 还主要体现在语言克制，不体现在可复核推导闭环**。

下面按优先级给出审稿意见。

---

## 一、必须优先修的硬伤

### 1. 交叉引用宏仍然有全局性机械错误，这是终稿级硬伤

**问题：**  
全文仍然存在大量 `图 图...`、`表 表...`、`式 式...` 的重复引用写法。这不是个别疏漏，而是宏或写法层面的系统性问题。

**为什么严重：**  
这会持续打断阅读，而且会直接削弱“教材可信度”。读者会下意识怀疑：如果交叉引用都没有清干净，那公式、编号、内部回指是否也可能有隐藏错误？

**建议改法：**
- 不要手工写“图/表/式 + `\ref`”；统一改成一套宏。
- 最稳妥的做法是：
  - 要么全部用 `\autoref` / `cleveref`；
  - 要么自己定义 `\figref{}`, `\tabref{}`, `\eqnref{}` 三个命令，然后全文替换。
- 这件事必须做**全局 regex 清洗 + 二次编译检查**，不要零散修。

---

### 2. 第二十六章仍然不是“完整算完的教材例题”，而更像“高质量执行模板”

**问题：**  
第二十六章比上一版进步很大，这是事实；但它**仍然没有真正完成**“fully worked example”的任务。  
你现在给出的主要是：

- 名义参数；
- 每一步应该建什么模型；
- 每一步应该输出什么量；
- 每一步能说什么、不能说什么；
- 最后整理成中间结果汇总表。

这非常好，但它仍然更像**标准答案格式**，而不是**真的带学生把数值闭环做完**。

**漏洞在哪里：**
- 例题 B 里真正关键的中间量没有被“实数化”。例如：
  - `n_eff`
  - `Γ_QW`
  - `Γ_loss`
  - 单胞候选模的频段
  - 上/下辐射比例
  - 有限阵列下主模/竞争模排序
  - 简化回写后的净模裕量变化
- 表 26.3 是“该产出什么”的表，不是“本题实际产出了什么”的表。

**为什么这对新手是致命缺口：**  
新手第一次做错，不是因为他不知道“要输出哪些量”，而是因为他不知道：
- 一个合理的数值大概是什么量级；
- 哪些数值一出来就说明模型设错了；
- 哪些图形态一出来就说明边界/PML/归一化出问题了。

**替代方案：**
- 保留现有“双层例题”结构；
- 但必须再加一个 **“例题 B 的实算结果页”**，至少给出：
  1. 一组具体 `n_eff, Γ_QW, Γ_loss`；
  2. 一张实际单胞候选模图/谱；
  3. 一张有限阵列近场 + 远场图；
  4. 一张简化 `J(r), N(r), T(r)` 分布示意；
  5. 一张“为什么这里仍不能宣称绝对 Ith / linewidth”的结论边界表。
- 你不需要伪造高保真器件数据；但你需要给出**教学型实算结果**。否则这一章仍然是“怎么做项目”，不是“我跟着你第一次就能做出来”。

### 3. 终端/命令行/开源可复现路径仍然明显不足

**问题：**  
你在第十三章给了一个真正可运行的 Python PWEM 脚本，这是很好的改进。  
但除此之外，整本书的主 workflow 仍然主要组织为：

- Lumerical 路径；
- COMSOL 路径。

对于“我将放在终端中运行”的目标读者，这仍然不够。

**前提不满足处：**
如果你的隐含读者是：
- 有商业软件；
- 接受 GUI 驱动工作流；
- 只需要 solver-agnostic 概念框架；

那么现在这版足够。  
但如果你的显式目标是：
- 能在终端运行；
- 想要快速搭建可复现研究流程；
- 希望尽量不依赖 GUI；

那么当前版本**前提不满足**。

**具体不足：**
- 只有第十三章有明确脚本与输出文件；
- RCWA / FDTD / FEM / 电学 / 热学部分没有形成对等的命令行入口；
- 没有给出最小目录结构、文件命名、参数清单格式、运行顺序、输出制品定义。

**替代方案：**
新增一个“命令行复现附录”，不需要展开软件教程，只要给最小骨架：
- `01_layers/`
- `02_unitcell/`
- `03_finite_array/`
- `04_electrical_thermal/`
- `05_postprocess/`
- `params.yaml`
- `run_stage_A.py ... run_stage_E.py`
- 每一步：
  - 输入文件
  - 输出文件
  - 必查 sanity checks
  - 失败后优先排查项

你不必承诺“全书都开源软件复现”，但必须给出一条**明确的 terminal-native 骨架**。否则这本书更像“理论 + 商业软件工作流导论”，不是“理论 + 仿真指引书”。

## 二、结构上仍然存在的问题

### 4. 阅读路径仍然没有真正分层，首读者还是会在第八至十二章掉速

**问题：**  
你现在在前言里已经提醒“第一次不要乱跳”，但这还不够。  
对新手最危险的区间仍然是：

- 第八章到第十二章；
- 第二十三、二十四章。

这些章节不是写错了，而是**数学密度与概念压缩度都偏高**。

**为什么这仍然是教材问题：**  
教材不是只要内容顺序正确就够了。教材还要控制第一次阅读的失速点。  
你现在的目录是平铺的 31 章，读者看不到“必须先吃下什么”“哪些可以第二遍回来再啃”。

**建议改法：**
至少做二选一：

**方案 A：目录层面加 Part**
- Part I 电磁骨架
- Part II 对称性 / BIC / 降阶理论
- Part III 数值方法
- Part IV 器件物理
- Part V 设计 / workflow / 失败模式
- Appendices

**方案 B：在章首页加阅读标签**
- `首轮必读`
- `二轮深化`
- `项目实施时回看`

我更推荐 A+B 都做。

### 5. 贯穿全书的 benchmark 仍然来得太晚

**问题：**  
第二十六章现在终于有了例题，这是进步。  
但**贯穿全书的单一 benchmark 结构**仍然没有真正建立。

**症状：**
- 前几章仍然主要在讲“抽象结构”“典型结构”“教学型结构”；
- 到第十三章有 PWEM 脚本；
- 到第二十六章才出现比较完整的例题。

这意味着读者前半本书一直在脑中换器件。

**为什么这会削弱教学效果：**  
新手最需要的是把“同一个器件”在不同层级被反复投影：
- Maxwell 层看它；
- Bloch 层看它；
- CWT 层看它；
- RCWA 层看它；
- 有限阵列层看它；
- 电热层看它。

如果 benchmark 直到后面才固定，前面的知识就更像“分章节学概念”，而不是“围绕一个对象逐层建模”。

**替代方案：**
从第二章或第四章开始就固定一个“本书 benchmark 器件”：
- 方格晶格 air-hole slab
- 一组名义层栈
- 一组名义工作波长
- 一组名义材料平台

之后每章都明确一句：
> 本章以下结果默认以 benchmark 结构为教学参照；若使用更一般结构，会在章内说明哪些结论仍成立。

## 三、pedagogy 仍然偏“研究型说明书”，还不够“本科自学教材”

### 6. 图示密度明显不够，尤其对一本 256 页左右的入门教材来说

**问题：**  
这本书现在的图比上一版更有针对性，但整体仍然**图太少、字太密**。  
对新手来说，图不是装饰，而是避免误读的结构支架。

**尤其缺图的地方：**
- 第三章：开放 Maxwell / QNM / 复频率平面
- 第七章：`γ, α, g_th, Γ_opt, Q` 之间的量纲链
- 第十章到第十二章：CWT 矩阵、TCMT 端口图、左/右模与伴随投影
- 第十四到第十六章：各求解器输入/输出对照图
- 第二十一到第二十六章：从单胞到有限阵列到电热回写的证据链图

**为什么这是硬需求：**  
你现在很多段落其实已经写得很清楚，但缺一张图就会让读者把 5 段文字的逻辑拆开来记。  
这种损失不是“读者多花一点时间”，而是**工作记忆直接超载**。

**替代方案：**
新增至少以下 8 类图：
1. `Q, γ, τ_p, α` 的关系图；
2. “封闭腔 vs 开放腔 vs QNM”的概念对照图；
3. CWT 四波基底到 irrep 基底的变换示意；
4. TCMT 端口-共振模框图；
5. 左模/右模/伴随投影的 2×2 非厄米玩具图；
6. 单胞筛选 → 有限阵列 → 电热回写 的证据链图；
7. 数据账本 / 参数流 / 文件流图；
8. 第二十六章 benchmark 的统一总图。

### 7. 你说自己“rigorous”，但当前 rigor 更多体现在态度上，未完全体现在推导闭环上

**问题：**  
这版最大优点之一，是你知道哪里不能乱下结论。  
但如果以“教科书标准”的 rigor 来衡量，当前的 rigor 仍然更多体现在：
- 严格表述框；
- 边界提醒；
- 适用条件警告；

而不是体现在：
- 完整证明；
- 可逐步复核的推导闭环；
- 明确区分定理/近似/工程公式。

**这不是吹毛求疵，而是一个真实风险：**  
读者会感到“作者很严谨”，但不一定能自己把式子推出去。  
这类书最容易产生一种假象：**理解了边界，不等于掌握了工具。**

**最需要补硬化的几个点：**

#### 7.1 第三章命题 3.1 的证明仍不够干净
**问题：**  
你在这里想说明“封闭、无损、无色散时 Maxwell 问题可写成自伴形式”，方向对。  
但当前写法把下面几件事压得太快：
- 这是广义本征值问题还是普通本征值问题；
- 采用的是哪种函数空间与约束子空间；
- `∇·(εE)=0` 约束在哪一步进入；
- 自伴性是对哪个算子、在哪个内积下成立。

**建议：**
- 不要把这里写成“看起来像一个完整定理证明”却没有把技术前提钉死；
- 更安全的写法是：
  - 正文给“物理上足够严格”的证明思路；
  - 附录给真正完整的算子形式与约束说明；
  - 并明确标注“这是教学化证明，不是函数分析层面的完整证明”。

#### 7.2 第七章阈值公式虽然正确，但还缺一个现场的单位桥
**问题：**  
`γ_opt = v_g g_mod`、`g_th = (α_rad + α_int)/Γ_opt` 这条链已经比上一版清楚。  
但对新手来说，最容易卡的仍然是：  
**这里的 `γ` 是 `s^-1`，`α` 是 `cm^-1` 还是 `m^-1`，`g` 又是什么单位？**

**建议：**
在第七章第一次出现这些量时，直接给一个小表：
- `γ`: `s^-1`
- `τ_p`: `s`
- `α`: `cm^-1`
- `g_mat, g_mod`: `cm^-1`
- `Γ_opt`: 无量纲
- `v_g`: `cm/s`

不要把这个任务留给读者去翻符号表。

#### 7.3 第八章/第九章的辐射选择规则还要更明确地区分“第一层判据”和“充分判据”
**问题：**  
你现在已经提醒过：in-plane irrep 只是最小规则，更严格的 3D slab 判断还需要纵向奇偶性与层栈对称。  
这是对的。  
但这句提醒**还不够凸显**，容易让新手把：
> `irrep 匹配`  
误读成  
> `因此该模一定能强辐射/一定不是 BIC`

**建议：**
在第八章和第九章都各加一句同样醒目的硬说明：
> `in-plane little-group 匹配是必要的第一层筛选，不是三维 slab 中法向辐射强度的充分判据。`

这样能避免读者把群论规则用过头。

#### 7.4 第十二章非厄米微扰已经抓住方向，但 এখনও缺一个 2×2 玩具模型
**问题：**  
你已经把左模/右模/伴随/双正交讲出来了，这很好。  
但对首次接触非厄米问题的读者，这一段仍然过于“抽象正确”。

**建议：**
在给出一般公式前，先给一个 2×2 非厄米矩阵：
- 写出右本征矢；
- 写出左本征矢；
- 说明为什么不能用右模自己的共轭直接投影；
- 再把它类比到开放 Maxwell。

这会比再加半页抽象文字更有效。

## 四、workflow 章节仍有“项目感很强、复现性不够硬”的问题

### 8. 第二十五章的数据账本是正确方向，但还不够可执行

**问题：**  
“先建数据账本，再建几何”这个原则是对的，而且很重要。  
但现在这部分仍然偏理念，没有落到一个研究者真的能拿来执行的 schema。

**目前缺的关键信息：**
- 参数唯一 ID
- 符号名 / 文件字段名 / 软件变量名的对应
- 单位
- 来源（测量 / 文献 / 拟合）
- 适用温度 / 频段
- 版本号
- 最后更新时间
- 被哪些 solver 使用
- 不确定度传播到哪些输出量

**建议：**
直接给一个最小 manifest 模板，比如：

```yaml
parameter:
  name: n_GaAs
  symbol: n
  unit: 1
  value: 3.42
  source: literature
  valid_wavelength_nm: [930, 990]
  valid_temperature_C: [20, 80]
  uncertainty: 0.02
  used_in:
    - stage_A_waveguide
    - stage_B_rcwa
  last_updated: 2026-03-01
